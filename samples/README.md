# Sample data

Synthetic data for trying SDSA. All values are fabricated ÔÇö any resemblance
to real people, accounts, or transactions is coincidental. `*.example`
domains are reserved for documentation use (RFC 2606).

### Small (for quick manual exercises)

| File                  | Format | Rows | What it exercises                                               |
| --------------------- | ------ | ---- | --------------------------------------------------------------- |
| `employees.csv`       | CSV    | 20   | HR-style PII: names, emails, phones, DOB, zip, salary           |
| `customers_cjk.csv`   | CSV    | 15   | Multilingual column names (Traditional Chinese) + UTF-8 data    |
| `transactions.csv`    | CSV    | 25   | Credit card numbers (Luhn-valid test BINs), amounts for DP      |
| `access_logs.txt`     | TXT    | 15   | Pipe-delimited logs                                             |
| `users.sql`           | SQL    | 10   | `INSERT` statements                                             |

### Large (realistic load, meaningful k-anonymity classes)

| File                       | Format | Rows | What it exercises                                                                        |
| -------------------------- | ------ | ---- | ---------------------------------------------------------------------------------------- |
| `employees_large.csv`      | CSV    | 1000 | 10 departments ├Ś 12 zips Ôćĺ natural equivalence classes, realistic salary distributions   |
| `transactions_large.csv`   | CSV    | 5000 | 300-user pool, 8 merchant categories, 5 countries, log-normal-ish amounts for DP         |
| `customers_large.csv`      | CSV    |  500 | Mixed Traditional Chinese + English names/addresses, 4 membership tiers                  |
| `access_logs_large.txt`    | TXT    | 3000 | 200-user pool, pipe-delimited, realistic status distribution                             |
| `users_large.sql`          | SQL    |  500 | Single-table `INSERT` dump with multi-row VALUES                                         |

Regenerate with: `python3 samples/generate.py` (deterministic, stdlib-only).

### Huge (load test)

| File                   | Format | Rows       | Size    | What it exercises                                      |
| ---------------------- | ------ | ---------: | ------: | ------------------------------------------------------ |
| `employees_huge.csv`   | CSV    | 1,680,000  | 200 MB  | Large-file streaming parse, full pipeline at scale     |

Not generated by default. Produce with:

```bash
python3 samples/generate.py --huge     # adds employees_huge.csv (~200 MB)
# or
python3 samples/generate.py --all      # generates everything
```

Observed end-to-end latency on an M-series Mac (local uvicorn):
- upload + schema/PII detect: ~0.5 s
- full pipeline (mask + hash + tokenize + truncate + DP Laplace + k-anonymity
  + validation + report): ~10 s

Backend default `max_upload_bytes` is 300 MB; override with
`SDSA_MAX_UPLOAD_BYTES` env var if you need something else.

## Suggested policies

### `employees.csv`
- `full_name` Ôćĺ `redact`
- `email` Ôćĺ `hash`
- `phone` Ôćĺ `mask` (keep country code)
- `dob` Ôćĺ `date_truncate` (year), mark as QI
- `zip` Ôćĺ `string_truncate` (keep 3), mark as QI
- `department` Ôćĺ `retain`, mark as QI
- `salary` Ôćĺ `dp_laplace` (lower 50000, upper 150000, ╬Á = 1.0)
- `hire_date` Ôćĺ `date_truncate` (month)

### `customers_cjk.csv`
- `ňžôňÉŹ` Ôćĺ `redact`
- `email` Ôćĺ `hash`
- `ÚŤ╗Ŕę▒` Ôćĺ `mask` (keep first 4 chars for country+area code)
- `ńŻĆňŁÇ` Ôćĺ `string_truncate` (keep 6 chars for city/district)
- `birthdate` Ôćĺ `date_truncate` (year), QI
- `membership_tier` Ôćĺ `retain`, QI
- `lifetime_value` Ôćĺ `dp_laplace` (lower 0, upper 100000, ╬Á = 0.5)

### `transactions.csv`
- `account_email` Ôćĺ `hash` (or `tokenize` for joinable output)
- `card_number` Ôćĺ `redact` or `mask`
- `amount_usd` Ôćĺ `dp_laplace` (lower 0, upper 5000, ╬Á = 1.0)
- `merchant_category` Ôćĺ `retain`, QI
- `country` Ôćĺ `retain`, QI

## All three formats are accepted

- **CSV** ÔÇö standard comma-separated.
- **TXT** ÔÇö delimiter auto-detected (tab, pipe, semicolon, comma).
- **SQL** ÔÇö `INSERT INTO ... VALUES (...), (...)` statements. Single-table
  dumps only; comments stripped; strings (with `''` and `\'` escapes),
  numbers, `NULL`, `TRUE`/`FALSE` supported. `CREATE TABLE` and DDL are
  ignored. Multi-table dumps are rejected ÔÇö split the file and upload one
  table at a time.
